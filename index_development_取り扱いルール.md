## 1. 基本運用ルール（本プロジェクト & 今後の全HTML系ソフト共通）

| ファイル | 位置付け | 公開範囲 | 更新タイミング |
|---|---|---|---|
| **index.html** | ある時点での完成版スナップショット。Webで知人に公開する版。 | 外部（GitHub Pages等） | 開発版で改良が一区切りついた時のみ手動で昇格 |
| **development.html** | 開発版。Gonti 個人が屋外・モバイルで閲覧しながら随時改修する版。 | 本人のみ | 随時（AI/手動いずれでも） |

運用フロー:

1. index.html を知人に公開 → 試してもらいフィードバック収集
2. 知人には見えないところで development.html を改修（機能追加・バグ修正）
3. development.html が一応の完成に達したと判断したら **更新ボタンで index.html に昇格**
4. 再び 1. に戻る

この運用を、今後の他ソフト（収集ソフト系、出船可否判断システム等）のHTMLにも水平展開する。

- `.promote-btn`: 「更新」ボタン。目立つ赤系（例 `#c62828`）。誤クリック防止のため少し小さめ。

### B. 確認モーダル（ポップアップ）

- ブラウザ標準の `confirm()` は使わず、**独自のモーダルダイアログ**を実装する（UI統一のため）。
- 中央表示、暗い半透明オーバーレイで背景を隠す。
- 表示内容:
  - タイトル: 「⚠ index.html 更新確認」
  - 本文: 「本当に更新しますか？」  
    補足: 「現在の development.html の内容を index.html として書き出します。この操作は元に戻せません。」
  - ボタン: 「実行」（赤） / 「中止」（グレー）

### C. 「実行」クリック時の動作

ブラウザはローカルファイルに直接書き込みできないため、以下の**2段階方式**を推奨:

#### 方式1（推奨・メイン）: ブラウザ内ダウンロード
1. `document.documentElement.outerHTML` で現在のHTMLを取得
2. `<!DOCTYPE html>` を前付けして Blob 化
3. `<a download="index.html">` をプログラム的にクリックしてダウンロード
4. ダウンロード後、案内メッセージを表示:
   > ダウンロードした index.html を
   > `fishing log 開発　釣果データ解析ソフト/釣果データ解析ソフト_v2/` フォルダに
   > **上書き保存**してください。その後、`sync_latest.bat` またはGitで push してください。

#### 方式2（補助・Windows側）: `promote_to_index.bat` の提供
サンドボックスからは書き込めないので、Windows 側で実行する .bat を併設:
```bat
@echo off
cd /d "%~dp0"
copy /Y "development.html" "index.html"
echo development.html を index.html に昇格しました。
pause
```
「更新」ボタンをクリックするのが面倒な時や、ダウンロードフォルダから手動コピーを避けたい時用。

方式1 と 方式2 のどちらを最終採用するかは実装時に相談して決める。  
→ **暫定案**: ボタンは方式1（ダウンロード）を実装し、.bat も並行提供する。

### D. アーキテクチャ方針：**埋め込み方式で統一**（2026-04-17 確定）

両HTMLのJSロジックは実は同じ（`window.ANALYSIS_DATA` があれば使う／無ければ `fetch('analysis_result.json')`）で、違いはHTML内にデータが埋め込まれているか否かだけ。今回の昇格後は **index.html も埋め込み型** に統一する。

採用理由:
1. **運用がシンプル** — ファイル1個をコピーすれば昇格完了。JSONとの同期ずれ事故がなくなる
2. **合計転送量が実は少ない** — 埋め込み版 1.76MB は fetch版 (HTML 78KB + JSON 2.3MB = 約2.38MB) より軽い
3. **file:// でも動く** — GitHub Pages 公開だけでなく、ZIP配布やローカルダブルクリックにも対応可能
4. **更新ボタン実装がシンプル** — `document.documentElement.outerHTML` を Blob 化してダウンロードするだけ

補足:
- `analyze_engine.py` が生成する `analysis_result.json` は**開発時の中間成果物として残す**が、公開版 index.html はこれを参照しない
- `analysis_result.json` を完全に廃止するかは将来判断（development.html での JSON フォールバックやデバッグ用途も兼ねるため当面は残す）

## 3. 作業ステップ（実行時）

1. development.html のバックアップ（`development_backup_YYYYMMDD.html`）
2. `<h1>` を改修（改良版バッジ + 更新ボタン）
3. CSS を追加（`.dev-badge`, `.promote-btn`, モーダル関連）
4. モーダル HTML を body に追加（初期 `display:none`）
5. JS 関数を追加: `showPromoteModal()`, `hidePromoteModal()`, `executePromote()`
6. 動作確認（ローカル file:// で「更新」→ダウンロードまで）
7. （オプション）`promote_to_index.bat` を同フォルダに配置
8. この運用ルール `.md` を `V2.1_取り扱い説明.md` に統合、または本ファイルを正とする
9. 動作OKなら development.html を（必要に応じて）index.html に昇格して GitHub へ push
